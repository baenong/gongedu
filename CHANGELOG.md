# Changelog

## v0.9.8

**AI 검증 제공자로 로컬 LLM(Ollama) 추가**

- OpenAI/Claude에 이어 사내 서버에서 직접 돌리는 로컬 LLM(Ollama)을 AI 검증 제공자로 선택 가능
- 한글 이수증 인식 성능을 고려해 Qwen2.5-VL 등 비전 모델 사용을 전제로 하며, 인증서 이미지가 외부 API로 전송되지 않고 서버 내부에서만 처리됨
- 모델을 교체해도 Ollama와 통신하는 코드는 그대로 두고 설정 화면에서 모델 이름만 바꾸면 되도록 구성
- 설정 화면에서 Local LLM 선택 시 API 키 입력 없이 서버에 설치된 Ollama 모델 목록을 조회해 선택 가능
- AI 설정 저장 API(`POST /settings/ai`)를 provider별로 흩어져 있던 모델/키 인자 대신 `provider`, `model`, `apiKey` 3개 인자로 단순화해, 새 제공자 추가 시 코드 변경 범위를 최소화

## v0.9.7

**기능개선 의견 작성자 표시 및 관리 기능 강화**

- 공개 의견 목록에 작성자 이름을 함께 표시 (부서 등 민감정보는 계속 제외)
- 본인이 작성한 의견은 직접 삭제 가능 (숨김 처리 — 관리자 이력에는 계속 남음)
- 총괄담당 이상은 타인의 의견도 숨김 처리하거나 완전 삭제(DB에서 영구 삭제) 가능
- 목록 정렬을 오래된 글이 위, 최신 글이 아래로 오도록 변경하고 새 글 작성 시 자동 스크롤 적용
- 기본 스크롤바 대신 위/아래 그라데이션 페이드로 추가 콘텐츠 존재를 표시

**AI 검증 튜닝을 위한 예시 교육명 등록**

- 교육과정 등록/수정 시 선택사항으로 "예시 교육명"을 여러 줄로 입력 가능 (등록된 과정명과 실제 이수증 명칭이 달라 AI가 관련성을 못 알아채는 문제 보완)
- 입력된 예시 교육명은 AI 검증 프롬프트에 그대로 전달되어 관련성 판단 기준으로 활용됨
- 부서담당 이상만 조회 가능, 교육담당 이상만 수정 가능 (일반직원에게는 노출되지 않음)

**교육 상세정보 모달 개선**

- 좌측(교육명/주관부서/마감일/상세정보/예시 교육명) · 우측(직원 이수 현황 표) 2단 레이아웃으로 재구성, 모달 너비 확장
- 표의 이름/부서/팀(계)/상태/AI 검증/관리 컬럼은 줄바꿈 없이 고정 너비로, 제출일/파일 컬럼은 말줄임(ellipsis) 처리
- 필터와 버튼(CSV/ZIP/삭제)을 분리해, 좁은 화면에서는 버튼 그룹이 필터 아래로 자연스럽게 줄바꿈되도록 개선
- 일반직원에게는 AI 검증/관리 컬럼을 숨기고 모달 크기도 축소해 불필요한 정보 노출을 줄임
- 모든 textarea에 기본 스크롤바 대신 위 피드백 목록과 동일한 hide-scrollbar + 페이드 효과 적용

## v0.9.6

**AI 기반 수료증 자동 검증**

수료증을 제출하면 AI가 파일 내용을 읽어 자동으로 검증합니다.

- 제출된 파일이 실제 수료증/이수증이 맞는지, 해당 교육과 관련이 있는지, 제출자 본인 명의가 맞는지 판단
- OpenAI, Claude 등 AI 제공자를 환경변수만으로 교체할 수 있도록 인터페이스를 분리 (PDF는 이미지로 변환 후 전달)
- AI 호출이 실패하거나 검증 결과가 의심스러워도 제출 자체는 항상 정상 처리되고, 문제가 있어 보이는 건만 관리자에게 경고 표시 (오탐으로 업무가 막히지 않도록 설계)
- 이수현황 화면에 AI 검증 상태(정상/의심/미검증) 컬럼과 필터를 추가, 부서담당 이상 권한자는 미검증 건을 직접 재검증 가능
- 제출 시 AI 검증을 백그라운드에서 수행하도록 변경해, 검증이 끝나길 기다리지 않고 제출 즉시 완료 화면을 볼 수 있도록 개선
- 판단 기준을 확장: 수료증/이수증 문구 포함 여부, 발급기관 명시 여부, 발급연도 일치 여부를 추가로 확인하고, 등록된 교육과정명과 글자 그대로 일치하지 않아도 주제가 같으면 관련 교육으로 인정하도록 개선 (예: "인권"을 주제로 한 교육 등록 시 "세계인권선언", "노인인권" 등도 인정)
- OpenAI는 요청마다 `store: false`를 지정하고, Claude(Anthropic)는 사용 중인 구조화된 출력 기능이 자동으로 Zero Data Retention(ZDR) 대상이 되도록 하여, 수료증에 포함된 개인정보가 AI 제공사 쪽에 불필요하게 남지 않도록 조치
- 설정 화면의 AI 검증 설정 접근 권한을 시스템관리자 전용에서 총괄담당 이상으로 확대

**이수현황 화면 개선**

- AI 검증 표시와 재검증·삭제·대신등록 버튼을 각각 별도 컬럼(`AI 검증`, `관리`)으로 분리해 가독성 향상
- 컬럼이 늘어난 만큼 상세정보 모달 너비를 확장

**기능개선 의견 기능 추가**

- 로그인한 사용자 누구나 하단 floating 버튼으로 의견을 남길 수 있음
- 다른 사용자가 남긴 의견을 작성자 이름과 함께 보고 좋아요를 누를 수 있음 (부서 등 민감정보는 제외)
- 본인이 작성한 의견은 직접 삭제할 수 있음
- 의견 관리 화면 접근 권한을 시스템관리자 전용에서 총괄담당 이상으로 확대, 총괄담당 이상은 타인의 의견도 숨김 처리하거나 완전 삭제 가능

**안정성 및 보안 개선**

- 수료증을 거의 동시에 두 번 제출하면(더블클릭 등) 중복 등록되던 경합 조건(race condition)을 수정
- 동일 건에 대한 AI 재검증이 겹치거나, 재검증 도중 삭제되어 결과가 유실되는 경합 조건을 수정
- 관리자용 이수현황 조회(`특정 사용자의 교육별 이수 현황`)에서 교육담당이 본인이 등록하지 않은 과정까지 조회할 수 있던 범위를 본인 등록 과정으로 제한
- 일반 설정 조회 API가 저장된 AI API 키 원문을 함께 반환하던 문제를 수정 (설정 화면은 키 존재 여부만 노출)

**테스트 환경 구축**

- 백엔드(Vitest + supertest), 프론트엔드(Vitest + React Testing Library) 모두 자동화 테스트 환경 도입
- 권한 스코프, AI 검증 판단 로직, 주요 화면(설정/이수현황 메인 화면 등)의 역할별 렌더링·API 호출 분기에 대한 테스트 커버리지 확대

**추가 보안·안정성 개선**

- `JWT_SECRET` 환경변수가 설정되지 않은 채로 기동되던 문제를 fail-fast로 수정 (미설정 시 서버가 즉시 종료)
- 비밀번호 변경 API에 무제한 시도 방지를 위한 rate limit 추가
- 사용자/부서 엑셀 업로드에 파일 크기 제한 추가
- 사용자·교육과정 수정 시 존재하지 않는 부서/팀 ID가 조용히 저장되거나 알 수 없는 서버 오류로 표시되던 문제를 수정해 명확한 오류 메시지로 응답
- `/settings/cleanup`의 SQL 조건절을 parameterized query로 변경
- 프론트엔드에 CSP 등 보안 헤더 추가, AI 설정 저장이 실패해도 입력했던 API 키가 화면에 남지 않도록 수정
- 이수현황 화면의 연도별 데이터 조회에 요청 취소(레이스 컨디션 방지) 적용, 데이터 조회 실패 시 사용자에게 오류 안내 표시
- Docker 배포 시 백엔드 컨테이너가 non-root 사용자로 실행되도록 변경

**리팩토링**

- 메인 화면(`MainPage`, 약 1400줄)을 교육과정 카드/등록 모달/상세·이수현황 모달 컴포넌트로 분리해 가독성과 테스트 용이성 개선
- 역할 판별 로직(팀계담당/부서담당/교육담당/총괄담당 이상 여부)을 공통 hook으로, 파일 다운로드 처리를 공통 유틸로 통합해 화면 간 중복 제거
- 사용자 관리, 설정 화면에 대한 테스트 커버리지 확대

- 부서·팀명이 길 때 `이수완료`/`미이수` 뱃지가 두 줄로 쪼개지던 문제 수정

## v0.9.3

**메인화면 캘린더 뷰 추가**

상단 필터 바에 "캘린더" / "교육목록" 탭을 추가해, 마감일 기준 월간 캘린더와 상세 카드 목록을 화면 전환으로 나눠 볼 수 있게 했습니다.
카드 목록만으로는 파악하기 어려웠던 마감일 분포를 캘린더로 한눈에 확인할 수 있습니다.

- 캘린더 탭: 좌측 캘린더 + 우측 간단 목록(상태색 점 + 교육명 (마감일))을 나란히 표시
- 교육목록 탭: 기존 상세 카드 그리드가 화면 전체 너비를 사용
- 날짜 칸에 마감 교육을 상태별(마감임박=빨강, 일반=주황, 이수완료=초록) 색상 박스로 표시하고, 화면 폭에 따라 표시 개수를 조정해 초과분은 "+N개 더보기"로 축약
- 캘린더의 날짜나 간단 목록의 교육명을 클릭하면 교육목록 탭으로 전환되면서 해당 카드로 스크롤 이동 후, 무지개색 빛이 테두리를 따라 흐르는 애니메이션으로 약 4초간 강조 표시

**엑셀 일괄 등록 양식 수정**

직원 엑셀 일괄 등록 양식이 실제 업로드 처리 로직과 열 개수가 맞지 않아 등록이 실패하던 문제를 수정했습니다.

**IP 화이트리스트 초기값 수정**

신규 설치 시 IP 대역 설정 초기값이 `127.0.0.1`로 들어가 외부 접속이 전부 차단되던 문제를 수정했습니다. 이제 초기값은 빈 값(전체 허용)입니다.

## v0.9.2

**Docker 배포 지원**

서버에 Node.js를 직접 설치하지 않아도 Docker만으로 실행할 수 있도록 배포 방식을 개선했습니다.

**SSH 터널을 통한 초기 설정 지원**

PuTTY 등 SSH 클라이언트로 서버에 원격 접속하는 환경에서도 `geadmin` 계정으로 로그인할 수 있도록 개선했습니다.
SSH 로컬 포트 포워딩을 설정하면 관리자 PC의 브라우저에서 서버 localhost로 접속할 수 있습니다.

## v0.9.1

**교육담당 ZIP 파일명 개선**

교육담당 권한으로 이수증을 ZIP으로 다운로드할 때, 파일명이 교육담당자 본인의 부서명으로 고정되던 문제를 수정했습니다.
이제 이수 현황 화면에서 설정한 필터에 따라 파일명이 자동으로 결정됩니다.

| 필터 상태               | ZIP 파일명 형식                |
| ----------------------- | ------------------------------ |
| 모든 부서 (필터 없음)   | `교육명_시간.zip`              |
| 특정 부서로 필터        | `[부서명]교육명_시간.zip`      |
| 특정 부서 + 팀으로 필터 | `[부서명]팀명_교육명_시간.zip` |
