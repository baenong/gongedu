# 메인화면 캘린더 뷰 추가 설계

## 배경

현재 메인화면(`frontend/src/pages/MainPage.tsx`)은 교육과정을 마감일 기준 정렬된 카드 목록으로만 보여준다. 마감일 분포를 한눈에 파악하기 어렵다는 의견이 있어, 카드 목록은 유지하되 캘린더 뷰를 함께 제공한다.

## 사용 환경

주 사용처는 사무실 PC(넓은 화면)이며, 모바일 접속도 발생한다. 기존 카드 목록은 이미 `md:grid-cols-1 lg:grid-cols-2`로 반응형 처리되어 있다.

## 레이아웃

- **데스크톱(넓은 화면)**: 캘린더를 좌측, 교육 카드 목록을 우측에 배치 (가로 분할)
- **모바일/좁은 화면**: 캘린더가 위, 카드 목록이 아래로 세로 스택
- 기존 필터 바(연도, 부서/팀, 미제출만 보기 등)는 그대로 상단에 유지하고, 캘린더와 목록 둘 다 같은 필터링 결과(`filteredCourses`)를 공유한다.

## 컴포넌트 구성

- 신규 컴포넌트 `frontend/src/components/CalendarView.tsx`를 만든다.
  - **입력**: 표시할 교육과정 목록(마감일, 이수 여부, 긴급도 포함), 현재 보고 있는 연/월, 월 변경 콜백, 날짜 클릭 콜백
  - **출력**: 월 그리드 UI. 별도 데이터 페칭 없음 — `MainPage`가 이미 들고 있는 `filteredCourses` 를 그대로 받아 렌더링만 담당한다.
- 외부 캘린더 라이브러리(FullCalendar, react-big-calendar 등)는 사용하지 않는다. 이 프로젝트는 `Badge`, `FormButton`, `Select` 등 UI를 전부 Tailwind 기반 커스텀 컴포넌트로 구성해왔고, 다크모드·커스텀 상태 배지·전용 상호작용(아래 참고)을 그대로 유지하려면 라이브러리를 얹는 것보다 직접 구현이 더 간단하고 일관적이다.
- 월 이동(이전/다음 달) 네비게이션을 캘린더 상단에 둔다. 초기값은 선택된 연도(`year` 필터) 내 현재 월(오늘이 해당 연도가 아니면 1월).

## 날짜 칸 UI

- iOS/Android 캘린더처럼, 마감일이 있는 교육을 그 날짜 칸 안에 **직사각형 색상 박스**로 쌓아서 표시한다.
- 박스 안 텍스트는 교육명이며, 칸 폭보다 길면 말줄임표(ellipsis)로 자른다.
- 색상/쌓는 순서(위 → 아래): **마감임박(빨강) → 일반(주황) → 이수완료(초록)** — 기존 카드 목록의 상태 색상 규칙(`MainPage.tsx`의 `isUrgent`/`isDone`/`isExpired` 판정)과 동일한 기준을 그대로 사용한다.
- 한 칸에 표시 가능한 최대 박스 개수를 넘으면 `+N개 더보기` 텍스트로 축약한다(칸 높이는 항상 고정 — 특정 날짜에 교육이 몰려도 그 주(row)의 칸 높이가 들쭉날쭉해지지 않는다).
- 최대 표시 개수는 JS로 실측하지 않고, Tailwind 반응형 브레이크포인트 구간별로 단계적으로 정한다(예: 좁은 화면 2개, 중간 3개, 넓은 화면 4개 — 정확한 구간값은 구현 단계에서 실제 셀 크기 보고 조정).

## 상호작용

- 날짜 칸(박스 포함 어디를 클릭해도 동일) 클릭 시: **우측/하단 카드 목록에서 해당 날짜가 마감인 교육으로 스크롤 이동 + 하이라이트**. 목록 자체를 필터링하지는 않는다(목록에는 계속 전체 필터링 결과가 보임).
- "+N개 더보기" 클릭도 동일하게 목록 스크롤/하이라이트로 처리한다(별도 팝업 없음) — 캘린더 칸 자체에 두 가지 다른 클릭 동작을 두지 않아 상호작용을 단순하게 유지한다.

## 범위 밖 (Out of scope)

- 캘린더에서 직접 교육 등록/수정하는 기능은 포함하지 않는다 (기존 "+ 교육과정 등록" 모달 흐름 유지).
- "다시 보지 않기" 등 온보딩성 안내는 없음 — 이번 스펙은 레이아웃/표시 로직에 한정.
- 캘린더 표시 여부를 껐다 켰다 하는 토글은 포함하지 않는다(항상 노출).

## 테스트 관점

- 월 경계(말일, 다음 달로 넘어가는 주) 렌더링이 올바른 날짜에 배치되는지
- 같은 날짜에 이벤트가 최대 개수를 초과할 때 "+N개 더보기" 개수 계산이 정확한지
- 날짜 클릭 시 목록 스크롤/하이라이트가 정확한 항목을 찾는지. 캘린더는 목록과 동일한 `filteredCourses`만 렌더링하므로, 필터에 의해 목록에서 제외된 교육이 캘린더에만 나타나는 경우는 구조적으로 발생하지 않는다.
